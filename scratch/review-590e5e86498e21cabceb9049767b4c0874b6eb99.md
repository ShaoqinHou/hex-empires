## Summary

Correctly migrates `crisisSystem` and `AchievementsPanel` away from `ALL_*` direct imports to `state.config` injection seams, and introduces two VII-spec constant modules (`CelebrationConstants`, `AdjacencyConstants`). No BLOCKs. Three WARNs: a hand-rolled LCG in `REDUCE_CITY_HAPPINESS` bypasses the canonical `nextRandom()` helper (pre-existing but unaddressed); `crises` in `GameConfig` uses `ReadonlyArray` causing O(n) scans where all other content types use `ReadonlyMap`; and `CELEBRATION_DURATION`'s explicit `Record<string, number>` annotation defeats `as const` narrowing. One NOTE: `resourceSystem` keeps its own local `CELEBRATION_DURATION_TURNS = 10` that duplicates the new module's value.

## Findings

### F-5b824310
- **severity:** warn
- **file:** `packages/engine/src/systems/crisisSystem.ts`
- **line:** 272
- **rule:** `engine-patterns.md` § "Seeded RNG — never Math.random()"
- **offender:** `const rngValue = ((state.rng.seed * 1664525 + state.rng.counter * 22695477 + 1013904223) >>> 0) / 0xFFFFFFFF;`
- **message:** Hand-rolls a different LCG formula directly on rng fields instead of calling `nextRandom(state.rng)`, producing a different distribution than the canonical mulberry32 in `SeededRng.ts` and bypassing the thread-through discipline.
- **suggested-fix:** Replace with `const { value: rngValue, rng: nextRng } = nextRandom(state.rng);` and return `{ ...state, rng: nextRng }`.
- **state:** open

### F-540bd44c
- **severity:** warn
- **file:** `packages/engine/src/types/GameConfig.ts`
- **line:** 46
- **rule:** `data-driven.md` § "Registry Pattern"
- **offender:** `readonly crises: ReadonlyArray<CrisisEventDef>;`
- **message:** Every other content type in `GameConfig` uses `ReadonlyMap<string, T>` enabling O(1) id lookup; `crises` uses `ReadonlyArray`, causing O(n) scans in `checkCrisisTriggers` and `resolveCrisis`.
- **suggested-fix:** Change to `ReadonlyMap<string, CrisisEventDef>`, update `GameConfigFactory`, and change `crisisSystem` to use `.get()`.
- **state:** open
