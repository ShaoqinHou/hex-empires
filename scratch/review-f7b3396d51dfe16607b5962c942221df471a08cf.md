## Summary

Phase 1.5.3 successfully tokenizes the majority of targeted raw-px inline-style values
across TopBar, BottomBar, PanelShell, and TooltipShell, introducing three well-justified
layout tokens (--chrome-button-sm-size, --chrome-bar-content-min-height, --border-hairline).
However, the commit's own acceptance criterion ("zero style={{ ... [0-9]+px hits in those
four files") is not fully met — `fontSize: '10px'`, `height: '32px'`, `height: '16px'`
remain in live JSX style props (with `// fixed:` justification comments), and the new
PROGRESS_TRACK_STYLE const encapsulates raw px values that avoid the JSX-literal test but
still embed undocumented magic numbers. --border-hairline was not applied to TooltipShell's
own border despite being the intended fix for exactly that pattern. No BLOCKs; commit can land.
