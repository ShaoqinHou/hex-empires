## Summary

This commit scaffolds the Independent Powers subsystem, Legends cross-session meta-progression
(mementos, Foundation/Leader Challenges), attributeSystem, and narrativeEventSystem. The
architecture is generally sound but has three BLOCK-level rule violations: `ageSystem.ts` imports
directly from `independentPowerSystem.ts` (forbidden cross-system import), and two system/state
files import `ALL_*` data globals instead of reading from `state.config.*`.

## Findings

### F-4642e094
- severity: block
- file: packages/engine/src/systems/ageSystem.ts
- line: 6
- rule: import-boundaries.md § "Systems cannot import from each other"
- offender: `import { createDefaultIPState } from './independentPowerSystem';`
- message: `ageSystem` directly imports a runtime function from `independentPowerSystem`, creating
  a forbidden cross-system dependency that breaks pipeline isolation.
- suggested-fix: Move `createDefaultIPState` to a shared utility — e.g.
  `packages/engine/src/state/IPStateFactory.ts` — and import from there in both systems.
- state: open

### F-9ec07c3b
- severity: block
- file: packages/engine/src/systems/legendsSystem.ts
- line: 23-25
- rule: engine-patterns.md § "state.config.X not ALL_X globals"
- offender: `import { ALL_FOUNDATION_CHALLENGES } from '../data/foundation-challenges'; import { ALL_LEADER_CHALLENGES } from '../data/leader-challenges'; import { ALL_MEMENTOS } from '../data/mementos';`
- message: `legendsSystem` imports three `ALL_*` data globals directly, bypassing the `state.config`
  injection seam; tests cannot inject focused fixtures and parallel test execution is broken.
- suggested-fix: Add `foundationChallenges`, `leaderChallenges`, and `mementos` registries to
  `GameConfig`, populate them in `GameConfigFactory`, and read them via `state.config.*` inside
  `evaluateLegends`.
- state: open

### F-31a01f60
- severity: block
- file: packages/engine/src/state/MementoApply.ts
- line: 12
- rule: engine-patterns.md § "state.config.X not ALL_X globals"
- offender: `import { ALL_MEMENTOS } from '../data/mementos';`
- message: `MementoApply` (a state utility called at game-start) imports `ALL_MEMENTOS` directly;
  tests cannot inject a trimmed memento set.
- suggested-fix: Thread the memento registry as a parameter or read from `state.config.mementos`
  once that field is added to `GameConfig`.
- state: open
