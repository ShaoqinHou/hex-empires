## Fix summary

**BLOCK F-36d8fce8** — `checkLegacyMilestones` didn't clamp per-axis tier gains to ≥0.

After `TRANSITION_AGE` resets `killsThisAge=0`, the first `END_TURN` could recompute
axis tiers lower than the previous age's value (e.g. `newPaths.military=0`,
`oldPaths.military=1` → gain=-1), silently decrementing `legacyPoints`,
`legacyPointsByAxis`, and `totalCareerLegacyPoints`.

**Fix applied in `packages/engine/src/systems/ageSystem.ts` `checkLegacyMilestones`:**
- Replaced bare `newPaths.X - oldPaths.X` with `Math.max(0, newPaths.X - oldPaths.X)` for all four axes (military, economic, science, culture).
- Introduced `clampedPaths` using `Math.max(oldPaths.X, newPaths.X)` per axis so the stored `legacyPaths` never regresses within an age.

**Tests:** 1711/1711 engine tests pass.
