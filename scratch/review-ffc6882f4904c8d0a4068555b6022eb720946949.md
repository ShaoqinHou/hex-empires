## Summary

This commit introduces the Narrative Events subsystem (10 events, 5 discoveries, a new system, and a panel). Engine logic and test coverage are solid. Two rule violations require remediation: a direct cross-system import from `movementSystem` into `narrativeEventSystem` (forbidden by `import-boundaries.md`), and a missing `ageGate` check in `evaluateAndEnqueue` that will cause antiquity-gated events to fire in later ages.

## Findings

### F-dbb59dc5
- severity: block
- file: packages/engine/src/systems/movementSystem.ts
- line: 5
- rule: import-boundaries.md § "Systems cannot import from each other"
- offender: `import { enqueueDiscoveryEvent } from './narrativeEventSystem';`
- message: `movementSystem` imports `enqueueDiscoveryEvent` directly from `narrativeEventSystem` — systems must be independent; shared logic belongs in a utility module under `state/` or `hex/`.
- suggested-fix: Extract `enqueueDiscoveryEvent` (and `isAgeGateOpen`) into `packages/engine/src/state/narrativeEventUtils.ts`, then import from that utility in both systems.
- state: open

### F-5d0e6027
- severity: block
- file: packages/engine/src/systems/narrativeEventSystem.ts
- line: 42
- rule: engine-patterns.md § "Age transition invariants" / data-driven.md § "age coherence"
- offender: `candidates = allEvents.filter(e => { if (fired.includes(e.id)) return false; if ((e.requirements.triggerType ?? 'END_TURN') !== 'END_TURN') return false; return matchesRequirements(e.requirements, player, state); });`
- message: `evaluateAndEnqueue` never calls `isAgeGateOpen`, so events with `ageGate: 'antiquity'` (`LOST_WANDERER`, `FIRST_BARBARIAN_VILLAGE`, `PLAGUE_OF_LOCUSTS`) will incorrectly fire during exploration and modern ages.
- suggested-fix: Insert `if (!isAgeGateOpen(e, state)) return false;` immediately after the triggerType check inside the candidates.filter predicate.
- state: open
