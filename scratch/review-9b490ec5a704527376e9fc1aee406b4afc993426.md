## Summary

The commit consolidates legacy-path scoring onto a predicate-driven `scoreLegacyPaths()` function
and adds golden-age cap (F-05), career totals (F-07), age-transition city downgrade, and per-age
kill tracking (F-09). Architecture is clean: no cross-system imports, no `Math.random()`, no live
state mutation. One BLOCK: `checkLegacyMilestones` does not clamp per-axis tier gains to >=0.
After `TRANSITION_AGE` resets `killsThisAge=0`, the very next `END_TURN` can compute negative
gains and silently decrement `legacyPoints`, `legacyPointsByAxis`, and `totalCareerLegacyPoints`.

## Findings

### F-36d8fce8
- severity: block
- file: packages/engine/src/systems/ageSystem.ts
- line: 373
- rule: engine-patterns.md § "Age transition invariants"
- offender: `const militaryGain = newPaths.military - oldPaths.military;`
- message: After `TRANSITION_AGE` sets `killsThisAge=0`, the first `END_TURN` re-evaluates
  `antiquity_military_t1 (killsThisAge >= 3)` as false. If `oldPaths.military` was 1 (earned
  in the previous age), `newPaths.military` becomes 0 -> `militaryGain = -1`. This subtracts
  from `legacyPoints`, `legacyPointsByAxis`, and `totalCareerLegacyPoints`, violating the
  F-07 career-total-never-decreases invariant. The same regression applies to any age/axis
  whose predicates use per-age counters that reset on transition.
- suggested-fix: Clamp each axis gain: `const militaryGain = Math.max(0, newPaths.military - oldPaths.military)` and also clamp the resulting paths: `military: Math.max(oldPaths.military, Math.min(3, axisTiers['military'])) as 0|1|2|3` so `legacyPaths` never regresses within an age.
- state: open

### F-4a84b534
- severity: warn
- file: packages/engine/src/state/LegacyPaths.ts
- line: 188
- rule: engine-patterns.md § "Seeded RNG" (proxy: logic correctness)
- offender: `(s.players.get(pid)?.killsThisAge ?? s.players.get(pid)?.totalKills ?? 0) >= 3`
- message: The `killsThisAge ?? totalKills` fallback is dead code in all normal post-transition
  turns: `??` only fires on null/undefined, but `killsThisAge` is set to `0` on TRANSITION_AGE,
  so `0 ?? totalKills` evaluates to `0`, never `totalKills`. The intended backward-compat shim
  (legacy saves without the field) works correctly, but creates false sense of safety.
- suggested-fix: Add comment: `// fallback only for legacy saves where killsThisAge is undefined; post-transition it is always 0 (not undefined)`
- state: open
