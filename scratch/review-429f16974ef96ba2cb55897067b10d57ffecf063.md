## Summary

Commit adds the codex system (Phase 1): per-tech `masteryEffect`/`masteryCodexCount` fields, `PLACE_CODEX` action, research-progress preservation via `techProgressMap`, and supporting `PlayerState`/`BuildingDef` type fields. The engine code is clean — pure functions, `state.config` seam, immutable patterns, no `Math.random()`. Two weak test assertions and one non-deterministic codex ID pattern in multiplayer scenarios are flagged.

## Findings

### F-56296ff5
- severity: warn
- file: packages/engine/src/systems/__tests__/researchSystem.test.ts
- line: 385
- rule: testing.md § "Assertion Rules"
- offender: `if (masteryBonus!.effect.type === 'MODIFY_YIELD') { expect(masteryBonus!.effect.yield).toBe('science'); ... }`
- message: The concrete yield/value assertions in the `'falls back to generic +1 science'` test are inside an unguarded `if` — if `masteryEffect` is ever non-MODIFY_YIELD, the inner expects silently skip and the test still passes.
- suggested-fix: Add `expect(masteryBonus!.effect.type).toBe('MODIFY_YIELD')` outside the `if` block (matching the pattern used in the `'applies per-tech masteryEffect'` test two blocks above).
- state: open

### F-aa2ccabb
- severity: note
- file: packages/engine/src/systems/researchSystem.ts
- line: 315
- rule: engine-patterns.md § "Seeded RNG — never Math.random()"
- offender: `` newCodexIds.push(`codex-${player.currentMastery}-${i}-${state.turn}`) ``
- message: Codex IDs encode tech + index + turn but not `playerId`; in a multiplayer game where two players master the same tech on the same turn, their codex IDs are identical — a correctness hazard if codex ownership is ever validated globally.
- suggested-fix: Include `player.id` in the ID: `` `codex-${player.currentMastery}-${player.id}-${i}-${state.turn}` ``.
- state: open

### F-476daff5
- severity: note
- file: packages/engine/src/systems/__tests__/researchSystem.test.ts
- line: 301
- rule: testing.md § "Assertion Rules"

- suggested-fix: Replace the `toBeDefined` + `if` block with a direct `toEqual` on the full effect shape.
