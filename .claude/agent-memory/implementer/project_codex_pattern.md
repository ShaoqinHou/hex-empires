description: How the codex system is structured in hex-empires engine (AA5.1)

The codex system uses two layers:

1. **PlayerState.ownedCodices** (existing, pre-AA5.1): simple array of codex ID strings.
   Used by PLACE_CODEX to assign codices to building slots for +2 science bonus.

2. **GameState.codices** (new, AA5.1): optional ReadonlyMap<string, CodexState> at the top level.
   Each CodexState has {id, playerId, cityId, buildingId, addedTurn}.
   Generated by researchSystem when a tech completes AND player has library/museum.

**Why:** The LegacyPaths predicate needed a codicesCount that counts state.codices
entries, rather than relying on codexPlacements (which tracks placement in slots).
These are distinct: codicesCount = earned, codexPlacements = placed.

The antiquity science legacy predicates now prefer codicesCount when state.codices
is present; fall back to scienceLegacyScore (codexPlacements*10 + techs) when absent.
